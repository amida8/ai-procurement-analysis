# 📊 Supplier KPI Dashboard（Python × Streamlit）

仕入先の納期・品質・リードタイムを可視化し、
リスクのある取引先を早期に把握するための
KPI ダッシュボード（Lv1）です。

---
### レベル設計
- Lv1：過去データを用いた KPI 可視化・ルールベース評価
- Lv2：時間推移分析（月別・トレンド）
- Lv3：予測・シミュレーション（遅延・品質リスク）

---
### 主な機能
- 仕入先ごとの調達数量（PCS）ランキング
- 48h 納期遵守率の可視化
- 品質（返修率）× スピード（E2E LT）の分布分析
- ルールベースによるリスク分類（LOW / MEDIUM / HIGH）

---
### 自動結論について
本アプリの結論は AI ではなく、以下のルールに基づく
シンプルな評価ロジックを採用しています。

例：
- 返修率 < 9%
- 48h 納期遵守率 ≥ 95%
- E2E リードタイム ≤ 全体平均

---
### データについて
- 本データは架空の仕入先データを想定したサンプルです
- 実務では ERP / CSV / Excel 出力データを想定しています

---

## 概要 / Overview

本プロジェクトは、  
**既存の仕入先（サプライヤー）KPIデータを Python で可視化・分析し、  
調達判断に役立つ示唆を得ること** を目的としたデータ分析作品です。

CSV / Excel ファイルをアップロードするだけで、  
仕入先の **納期・品質・取引規模** を即座に可視化できる  
シンプルなダッシュボードを実装しています。

📌 本プロジェクトは、  
**「仕入先KPIをもとに、どの取引先を優先・注意すべきかを即判断できる」  
業務向けダッシュボード** です。

---

## 🎯 目的 / Objective

- 既に存在する仕入先データを活用し  
- Python を用いてデータを構造化・可視化し  
- 調達・仕入判断の補助となる情報を提供する  
本ダッシュボードは、  
属人的になりがちな仕入判断を  
可視化・共通認識化することを目的としています。

👉 **分析のための分析ではなく、実務利用を想定した設計です。**

---

## 🗂 データ内容 / Data Description

本ダッシュボードでは、以下の指標を使用します。

| カラム名 | 内容 |
|---|---|
| supplier | 仕入先名 |
| pcs | 当月の調達数量（PCS） |
| on_time_48h | 副資材調達工程における48時間以内の納期遵守率（％） |
| bulk_lead_time_days | 注文から製品を顧客へ納品するまでの全体リードタイム（日） |
| return_rate | 当月の不良率（％） |

※ `on_time_48h` は **全体納期ではなく、副資材調達分岐工程のみ** を対象としています。

---

## 📈 可視化内容 / Visualization

ダッシュボードでは以下を表示します。
1. PCS（調達数量）ランキング  
   - 取引規模の大きい仕入先を把握するため
2. 48h 納期遵守率ランキング（副資材工程）  
   - 初動対応の安定性を比較するため
3. **品質 × スピード 分析**  
   - 不良率（品質）  
   - 大貨リードタイム（スピード）  
4. **リスク分類（Lv1 ルール）**  
   - LOW / MEDIUM / HIGH  

---
## 🧠 自動分析 / Auto Insights（Lv1）

以下の業務ルールにより自動判定しています。

- **優先候補（Top3）**
  - PCS が高い
  - 納期遵守率 ≥ 95%
  - 不良率 ≤ 3%

- **要注意仕入先（HIGH）**
  - 納期遵守率 < 90%
  - または 不良率 > 5%
※ 各しきい値はサンプルデータを想定した例であり、
実務では業種・商材・取引条件に応じて調整される前提です。
※ 本ルールは、  
実際の現場ヒアリングを想定した  
**シンプルかつ説明可能な基準** としています。
